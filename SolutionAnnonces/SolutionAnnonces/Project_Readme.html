Ajouter des fonctionnalités d'appareil (plug-ins)

Apache Cordova utilise des plug-ins pour fournir un accès aux fonctionnalités d'appareil natives qui ne sont pas accessibles aux applications web simples. Parmi ces fonctionnalités, citons le système de fichiers, l'appareil photo, l'accéléromètre et les contacts de l'appareil.

Pour ajouter des fonctionnalités d'appareil, ouvrez config.xml à la racine de votre projet. Cette action lance le gestionnaire de configuration d'application dans lequel vous trouverez un onglet Plug-ins qui propose des plug-ins de base validés par Microsoft. Vous pouvez également ajouter des plug-ins personnalisés parmi les centaines proposés par la communauté.

Essayez d'ajouter ce plug-in de test personnalisé maintenant : https://github.com/MSOpenTech/cordova-plugin-test

Utiliser le code que vous connaissez

Si vous savez générer des applications web, vous serez rapidement en mesure de générer des applications mobiles pour iOS, Android et Windows à l'aide d'Apache Cordova. En outre, vous pouvez utiliser votre infrastructure JavaScript favorite lors de la génération de votre application (par exemple, Angular, React, Ionic).

Essayez-le maintenant en ajoutant votre infrastructure JavaScript préférée au dossier de scripts et en modifiant le code de votre application.

Générer et exécuter (sur les émulateurs et les appareils)

Vous pouvez générer et déployer votre application sur des appareils et des émulateurs exécutant Android, iOS, Windows et Windows Phone.

Au début, la plupart des développeurs préfèrent travailler dans le navigateur. L'émulateur Ripple, qui s'exécute dans Chrome, constitue la cible de déploiement par défaut pour les nouveaux projets. Pour l'essayer, sélectionnez Ripple et appuyez sur le bouton de lecture dans la barre d'outils. Chrome doit alors s'ouvrir et charger votre application.
